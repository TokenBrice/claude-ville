# v0.24 — Reactive Sound Atmosphere

Status: `ready` · Baseline: v0.23.0 (`d31f693`) · Owner paths: `claudeville/src/presentation/shared/audio/` (new), `shared/AmbientAudioController.js`, `shared/TopBar.js`, `claudeville/index.html`, `character-mode/IsometricRenderer.js` (one emit), `character-mode/WeatherRenderer.js` (one emit), `CHANGELOG.md`, `package.json`.

## 1. Diagnosis — why the current sound is "noise, not atmosphere"

Current implementation is a single file, `claudeville/src/presentation/shared/AmbientAudioController.js` (~430 lines), self-described as `mode: 'simple-ambient'`:

1. **The "air" layer literally is noise.** Band-passed white noise at 620 Hz (plus a 1900 Hz "shimmer" tap) runs continuously. At those frequencies filtered noise reads as tape hiss, not wind.
2. **The pad never changes.** Four raw detuned oscillators (A2/E3/A3/B3, sine/triangle) drone forever with only a 0.035 Hz amplitude swell. Static spectra cause listening fatigue within minutes; raw oscillator timbre sounds synthetic rather than village-like.
3. **Chimes are too frequent and aimless.** A random sine note fires every 5.8–12 s (~7 notes/min, forever), with no phrase structure and no rests. Good ambience breathes; this never does.
4. **Everything plays at once, always.** All three layers run at fixed gains from the moment sound is enabled until it is disabled. There is no silence, no dynamics, no variation across an hour of listening.
5. **Zero reactivity.** It subscribes to nothing — no atmosphere, no eventBus, no village state. Meanwhile the visual world models dawn/dusk, six weather types, seasons, storms with lightning, arrivals, incidents, councils, and harbor traffic.
6. **No volume control.** Master gain is hard-coded to 0.38; the only control is on/off.

History note: v0.12 played a cue per event (`agent:added`/`agent:updated`/`usage:updated` → wall of bleeps); commit `6368871` overcorrected to this static drone. v0.24 builds the missing middle: **continuous layers driven by world state + rare, governed one-shot cues**.

## 2. Design principles

- **Silence-first.** The default (clear day, quiet village) is *near-silence*: faint wind, occasional birdsong. Layers rise only when the world justifies them. Rests are part of the music.
- **The village is the score.** Continuous layers track continuous state (phase, weather intensity, working-agent count); one-shot cues track discrete moments (arrival, distress, council, hour) behind a strict global governor — the v0.12 lesson, centralized this time.
- **Stay procedural, but craft it.** No samples, no assets, no build step (keeps the v0.13 promise and the repo's zero-build architecture). Quality comes from shaping: gusting wind envelopes, droplet grains, FM chirps, one shared tonal scale.
- **One key, one mix bus.** Every pitched element (bed, motif, cues) draws from a single pentatonic scale whose register/brightness follows the phase, so nothing ever clashes. All ambience passes through one bus that cues duck against.
- **Keep the working lifecycle.** Opt-in default-off, gesture unlock, tab-hidden suspend, `localStorage` persistence, and button rendering in the current controller are correct — preserve them verbatim.

**Atmosphere delivery:** `IsometricRenderer` already calls `atmosphereState.update()` per frame. Add a throttled `eventBus.emit('atmosphere:updated', snapshot)` at ≤1 Hz (reuse its existing ~2/s fps-emit cadence point). The audio director consumes that event so debug overrides (`__claudeVilleAtmosphere.setHour/setWeather`) and village weather influence flow into audio for free. Fallback: if no event arrives for 3 s (Dashboard mode with World loop stopped), the director computes its own `createAtmosphereSnapshot()` — the module is pure local-clock, so ambience keeps tracking time/weather in Dashboard mode too.

## 4. Architecture

New directory `claudeville/src/presentation/shared/audio/` (vanilla ES modules, no deps):

```
audio/
  AudioEngine.js      // context + unlock + master chain + shared DSP helpers
  AudioDirector.js    // 1 Hz brain: world state → layer intensities + cue triggers
  MusicalScale.js     // phase → tonal center + pentatonic note table (shared by bed, motif, cues)
  CueGovernor.js      // global rate limiter: per-kind cooldowns + max cues/min budget
  layers/WindLayer.js
  layers/RainLayer.js
  layers/BirdsLayer.js
  layers/CricketsLayer.js
  layers/VillageHumLayer.js
  layers/TonalBedLayer.js
  layers/MusicLayer.js
  cues/CueKit.js      // one-shot synth voices (bells, motifs, thunder)
```

`AmbientAudioController.js` stays as the facade TopBar already owns: button, unlock gesture, visibility handling, enable persistence. On activate it constructs `AudioEngine` + `AudioDirector`; on deactivate it tears them down. `TopBar.js` keeps `new AmbientAudioController({ button })` — only gains the volume slider element.

### AudioEngine

- Context lifecycle (moved from the current controller: resume/suspend, unlock).
- Master chain: `ambienceBus (gain) → masterGain (volume) → gentle lowpass → compressor (safety only) → destination`; separate `cueBus → masterGain`.
- Volume: 0–1 slider value persisted at `claudeville.sound.volume` (default 0.5), mapped perceptually (`gain = v²`).
- Ducking: `duck(amount, holdMs)` lowers `ambienceBus` by ~2.5 dB with 80 ms attack / 1.5 s release; CueKit calls it per cue.
- Shared helpers: cached noise buffers (white + pre-filtered brown), `envGain()`, scheduled-ramp utilities. All layer/transient node cleanup patterns carry over from the current file (they are sound).

### AudioDirector

- `setInterval` tick at 1 Hz (audio needs no rAF; no per-frame work).
- Inputs per tick: latest `atmosphere:updated` snapshot (or own fallback snapshot), `world.getStats()`, season token, and counters accumulated from event subscriptions since last tick.
- Output: a target intensity 0–1 per layer, applied via `setTargetAtTime` with long time constants (2–6 s) — **no audible steps, ever**. Weather already provides smooth `intensity`/`precipitation` interpolation; the director just maps it.
- Event subscriptions (`village:scene`, `distress:watchtower`, `team:gather`, `chronicle:aurora`) forward to CueGovernor → CueKit.
- Exposes a debug snapshot for `window.__claudevilleAudio()`.

### Layer spec (mapping table)

| Layer | Recipe (all Web Audio primitives) | Intensity function |
| --- | --- | --- |
| **Wind** | brown-ish noise → lowpass 300–900 Hz; slow gust LFO (0.05–0.2 Hz) modulating gain + filter cutoff; gust rate/depth scale with wind | `0.06 + weather.intensity·0.5`; cutoff and gust depth from `|windX|`; fog adds low, muffled character |
| **Rain** | white noise → highshelf −∞ below 1.5 kHz shaped by precipitation; plus sparse "droplet" grains (short filtered blips, density ∝ precipitation); storm adds a 60–110 Hz rumble bed | `weather.precipitation` (0 in clear/fog); winter (snow visuals) mutes rain to near-zero and lets wind carry the scene |
| **Thunder** | one-shot: burst of lowpassed brown noise, 2–5 s decay, slight pitch fall | fires on storm-flash event (see §5), delayed 300–1200 ms after the visual flash; storm without World mode running uses its own Poisson timer ∝ intensity |
| **Birds** | 2-op FM chirps, 2–5 note phrases, randomized inter-phrase rest 8–40 s | peak at dawn (`phase==='dawn'`), low during day, 0 at dusk/night; ×(1−precipitation), ×(1−intensity·0.5); winter ≈ ×0.25 |
| **Crickets** | band-passed pulse train ~4.2 kHz, 2 alternating "individuals" with different rhythms | night only, ramp in via `phaseProgress`; summer 1.0, spring/autumn 0.5, winter 0; rain suppresses |
| **Village hum** | very quiet: occasional soft wood-tap/anvil-tick grains + low murmur (filtered noise formants), density ∝ working count | `clamp(stats.working / 6)`; 0 when nobody works; suppressed at night (village sleeps even if agents don't) |
| **Tonal bed** | replaces the pad: 2 voices max, heavily lowpassed, slow attack/release swells (20–40 s) on scale tones, long silences between swells | brightness/register from phase (warm low at dusk, airy at dawn, dark and sparse at night, nearly absent mid-day); overall gain ≤ half the current pad |
| **Music motif** | replaces chimes: short 3–6 note generative phrase on the shared scale, soft triangle-through-lowpass voice + existing delay/feedback network (keep that graph, it's good) | plays one phrase then rests **30–120 s** (vs 6–12 s today); phrase density nudged up slightly by village activity, down at night |

### CueKit + CueGovernor

One-shot voices, all on the shared scale, all through `cueBus`:

| Cue | Trigger | Sound | Per-kind cooldown |
| --- | --- | --- | --- |
| Arrival | `village:scene` kind arrival | two ascending scale notes | 20 s |
| Departure | scene departure | two descending notes | 20 s |
| Distress | `distress:watchtower` errored/rate_limited | single low muted bell | 30 s |
| Recovery | watchtower recovered | soft resolving third | 30 s |
| Council | `team:gather` | three-bell pattern | 60 s |
| Hour bell | director tick sees `clock.minutes === 0` | one bell strike (08:00–20:00 only) | 55 min |
| Aurora | `chronicle:aurora` | slow shimmer gliss | 120 s |

Governor on top of per-kind cooldowns: **global budget of max 6 cues/min and min 4 s spacing**; excess cues are dropped (not queued). A busy fleet session must sound like a lively village, not a switchboard.

## 7. Risks / decisions taken

- **Procedural, not samples** — chosen to keep the zero-asset promise and avoid a licensing/pipeline question. If phase-2 wind/rain can't be made pleasant purely procedurally, the fallback is a small set of self-generated looped noise buffers rendered offline into the repo — decide only after hearing phase 2, and flag to the operator first.
- **Cross-mode ambience** — ambience keeps playing in Dashboard mode (atmosphere is mode-independent); only the flash-synced thunder degrades to a timer. If this feels wrong in use, gating on `mode:changed` is a 5-line director change.
- **`prefers-reduced-motion` does not gate audio** (it is a visual-motion preference); the cue governor is the audio analog of the motion budget. Sound remains opt-in and off by default, which is the stronger accessibility position.
- **CPU** — worst case ≈ 12–18 concurrent nodes (vs ~15 today); the 1 Hz director is negligible next to the render loop. No `CanvasBudget` interaction.
