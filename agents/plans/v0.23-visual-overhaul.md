## Workstream A — Harbor fleet restoration (headline feature)

**Files:** `claudeville/src/presentation/character-mode/HarborTraffic.js` only.

Archaeology (`46b4e80` created one-ship-per-commit; `06e9f4c` capped visible ships per repo; `4d6ba13` added commit-count ship classes; `e9a1831` added stack sprites): the state reducer **still creates one ship per commit**. Consolidation is purely visual — `buildDockedVisualPackMap` (~1655) collapses ≥5 docked ships per repo/zone into one drawable, and `harborShipPackSize`/`distributedHarborShipPackSize` (~1734–1768) inflate the lead ship into an N-commit class hull. Restoration is a rendering-layer change.

| # | Item | Effort/Impact |
| --- | --- | --- |
| A1 | **Restore one-visible-ship-per-commit + flock departure.** Docked ships render individually up to ~15 per repo (raise `HARBOR_DOCKED_VISUAL_PACK_THRESHOLD` 5→15); stack sprites only as overflow past the cap. Departing ships always pack size 1 → a 12-commit push sails as a 12-ship flock. | medium / **high** |
| A2 | **Flagship leader hybrid.** The lead ship (`repoDockIndex===0`) keeps its commit-count class (grows with fleet size, carries repo heraldry); all other commits are individual skiffs. On push the flagship leads the flock (release-convoy line already links leader→followers). | small / medium |
| A3 | **Docked bob + sailing heave.** ±1px sine bob and tiny roll on the hull itself (`_drawShipSprite` ~3856), phase-seeded per ship id so the flock bobs out of sync; badges/crates/flags ride the same offset. Static under reduced motion. | small / **high** |
| A4 | **Waving repo flags & pennants.** Two-segment procedural wave on `_drawRepoFlag` (~4432) and the commit pennant; departing ships stream harder. | small / medium |

| A7 | **Fleet-count banner on the lead ship** (`N⚓` in repo accent) replacing scattered per-ship tier badges for packSize-1 skiffs. | small / medium |

## Workstream D — Agent accessories, helmets & hats

**Files:** `Compositor.js`, `AgentSprite.js`, `shared/RoleAccessory.js`.

Root causes found: `_compositeAccessory` (Compositor.js:86–113) stamps the same overlay at ONE fixed anchor (centerX, `0.22*cellSize`) for all 80 cells while the real head apex moves up to 12px vertically / 13px horizontally across directions and walk frames; accessory choice keys on `currentTool` so hats teleport on/off; baked hat pixels inflate content bounds so the whole body shrinks when a hat appears.

| # | Item | Effort/Impact |
| --- | --- | --- |
| D1 | **Per-cell head-apex anchoring.** Scan each cell's topmost opaque rows (alpha-weighted centroid) and stamp the overlay at the measured apex — hats track facing direction and walk bob automatically. Composite-time only, zero per-frame cost. | small / **high** |
| D2 | **Accessory hysteresis.** Keep the current accessory until a different candidate has been dominant ≥20s; stops mid-stride hat teleporting. | small / **high** |
| D3 | **Exclude accessory from body draw-scale.** Compute `_spriteDrawScale` bounds from the accessory-free base so villagers stop shrinking ~6% when a hat appears. | small / medium |
| D4 | **Contact shadow + palette harmonization.** Darken 1–2 head pixels under the accessory's opaque bottom edge; low-alpha tint of the overlay toward the provider palette trim so hats stop reading as stickers. | medium / medium |
| D5 | **Back-facing treatment.** For n/ne/nw cells draw the top portion only, +1px down, darkened ~12%, so goggle lenses/hood openings stop appearing on the back of heads. | medium / medium |

**Cut (deferred):** pre-baked weapon transforms; true 8-direction accessory sheets (new assets — revisit only if D1–D5 insufficient).
